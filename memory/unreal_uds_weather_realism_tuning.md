2026-07-02 (session ac84b74e). Follow-up to [[unreal_uds_weather_panel_progress]]. User report: clicking Rain/Snow buttons spawns particles (so `ChangeWeather` + `BP_UDW` work), BUT the sky/atmosphere/lighting does NOT change — stays sunny; Cloudy button does nothing visible.

**ROOT CAUSE (confirmed empirically):** The UDS weather presets (`/Game/UltraDynamicSky/Blueprints/Weather_Effects/Weather_Presets/*`, class `UDS_Weather_Settings_C`) had far-too-mild `Cloud Coverage` values relative to the level's baseline (~3.808). Original: Clear_Skies CC=0/Fog=0, Cloudy CC=5/Fog=1, Overcast CC=7.5/Fog=1.5, Rain CC=7.5/Fog=3/Rain=7, Snow CC=8.5/Fog=5/Snow=6. So Rain only nudged clouds 3.8→7.5 = barely visible → "stays sunny." Particles (Rain=7/Snow=6) applied fine, decoupled visually from the weak cloud change.

**KEY SCALE FACT:** UDS `Cloud Coverage` on THIS project is NOT 0–1 (a research agent wrongly claimed 0–1). Empirical scale ~0–20+: baseline 3.8 = light scattered, 9 = clearly cloudy, 15 = overcast/rainy grey (city still visible), 17 = heavy flat overcast, 18/Fog6 = too foggy (horizon lost). `Fog` prop ~0–6 useful range. Verified by setting UDW `Cloud Coverage`/`Fog` directly in EDITOR (PostEditChange updates the sky live) + CaptureViewport.

**FIX APPLIED (presets edited via ObjectTools.set_properties on `<preset>.<preset>` + saved via save_assets):** Clear_Skies CC=1.5/Fog=0, Cloudy CC=9/Fog=1.5, Overcast CC=17/Fog=3.5, Rain CC=15/Fog=5 (Rain=7 kept), Snow CC=13/Fog=4 (Snow=6 kept). Verified each look in-editor by simulating on the level UDW + capture: Cloudy=nice grey partly-cloudy, Overcast=dark flat, Rain(15/4)=moody overcast rainy — all clearly distinct from sunny. NOTE presets are shared UDS plugin content (affect all 3 levels; may be overwritten on a UDS plugin update).

**REAL ROOT CAUSE FOUND & FIXED 2026-07-03 (session c33362b2):** The "particles change but sky/clouds/sun don't" bug was NOT the presets — it was the **UDW actor's Manual Override flags**. On `Ultra_Dynamic_Weather_C_0` in Glavniy_MirMIron, `cloud Coverage - Manual Override`=TRUE and `fog - Manual Override`=TRUE (rain/snow/wind overrides were false). Per UDS docs, Manual Override "is applied at the very END of generating the local weather, so it takes PRIORITY over all other sources of state" (incl. ChangeWeather). So ChangeWeather set CC=15/Fog=5 but the override forced them back to the manual (clear) value every frame → sky stayed sunny; Cloud/Overcast buttons (which ONLY change CC+fog, no particles) looked totally dead; Rain/Snow still showed particles (those overrides were off). **Self-inflicted:** I set these overrides ON in the prior session while previewing CC/Fog directly on the UDW in-editor and left them on. FIX: `ObjectTools.set_properties` on the UDW actor → `cloud Coverage - Manual Override`=false, `fog - Manual Override`=false; saved via `save_assets(["/Game/Level/Odilbek/Glavniy_MirMIron"])` (save_actor fails on level-embedded actors → must save the level asset). Override property keys live on UDW: `<name> - Manual Override` bools (cloud Coverage/fog/rain/snow/dust/wind Intensity/thunder-Lightning/material Wetness/material Snow Coverage/material Dust Coverage). **STILL TODO: check Pragulka + VR level UDW actors for the same stuck overrides** (likely also on if previewed there; each level must be loaded + its UDW fixed + level saved separately). Widget graph itself is CORRECT & unchanged — weather buttons OnClicked_Event_12..15 → `ChangeWeather(preset,3.0)` on cached `BP_UDW`; Winter OnClicked_Event_18 = SetSeason(3)+ChangeWeather(Snow). UDW↔UDS link verified: UDW prop `ultraDynamicSky` = Ultra_Dynamic_Sky_C_0. The earlier ×60→×100 TIME fix did NOT cause this (unrelated). EventTick change-detection is correct (Branch then→SetLastSlider→ApplyTime; only fires on slider change, not every frame).

**SEASON BUTTONS ROOT CAUSE (2026-07-03, session c33362b2):** Summer/Autumn buttons "do nothing" for TWO reasons: (1) **`season Mode` = "Use UDS Date"** on UDW — season is auto-computed from the in-game date every tick, so widget `SetSeason(1/2/3)` is immediately overwritten. FIX = set UDW `season Mode` = **"Manual Setting"** (enum `UDS_SeasonMode` has exactly 2 values: "Use UDS Date" | "Manual Setting"). Then SetSeason sticks. (2) Season float alone does NOT change cloud coverage/sky — it drives sun seasonal declination (sun height/day length), foliage color (if materials use UDS seasonal material funcs), temperature, and random-weather probability. So a season button must ALSO set matching weather/color for a visible atmosphere change (Winter already does SetSeason+Snow). Current UDW state read live: `season`=3.78, `apply Climate Preset`=None, `individual Seasons`=[0.78,0,0,0.22], `meteorological Seasons`=true. The `apply Climate Preset` + season probability maps are UDS's official seasonal-WEATHER mechanism (climate preset per region sets seasonal weather probabilities for Random Weather).

**EXPOSURE / PPV (task 4/5, 2026-07-03):** UDS Sky actor master toggle **`apply Exposure Settings` = TRUE** (currently) — while true, UDS drives PostProcess exposure (autoExposure*/localExposure* props + per-weather `exposure Bias Day/Cloudy/Night/Foggy/Dusty/Dawn-Dusk`) and OVERRIDES any PPV exposure. `exposure Metering Mode`=AEM_Histogram. To let a PPV control exposure, set `apply Exposure Settings`=false (tradeoff: lose UDS auto day/night exposure adaptation — decide per research). Sky-actor exposure prop names verified live: `apply Exposure Settings`, `exposure Metering Mode`, `exposure Bias <state>`, plus full autoExposure*/localExposure* set with bOverride_* flags.

**Cloud/Overcast preset re-tune 2026-07-03:** user said Cloud≈Overcast (both too heavy). Lowered Cloudy CC=9→5, Fog=1.5→0.5 (sunny+light cloud); Overcast kept CC=17/Fog=3.5; Partly_Cloudy tuned to CC=7/Fog=2/wind=3 for Autumn use. All saved.
